From Logic Gates to FPGAs: Schematic to Verilog

By SQGATE Engineering Team | Published on July 2, 2026 | 10 min read

In the early days of digital electronics, engineers designed circuits by drawing intricate schematics on vast sheets of paper. Each AND gate, OR gate, and flip-flop was meticulously placed and routed. As designs grew from hundreds of gates to millions and eventually billions, this visual paradigm became completely unsustainable. Enter Hardware Description Languages (HDLs) like Verilog and VHDL. Today, modern digital design for FPGAs (Field Programmable Gate Arrays) and ASICs relies entirely on these textual representations.

The Paradigm Shift: Space vs. Time

For software engineers transitioning to hardware design, the shift from imperative programming (like C++ or JavaScript) to HDL is often the hardest hurdle. In software, code executes sequentially—one line after the other. In Verilog, the code describes physical hardware. When you write a line of Verilog, you are instantiating physical logic gates or describing concurrent signal flows that all happen simultaneously.

A schematic fundamentally represents spatial connections. A wire drawn from an output pin to an input pin is a permanent, physical pathway for electrons. Verilog captures this same spatial reality textually. The key realization is that an FPGA evaluates all combinational logic paths concurrently. There is no "program counter" stepping through your assign statements.

Translating Combinational Logic

Combinational logic refers to circuits where the output is a pure function of the current inputs, with no memory or state. Translating a combinational schematic into Verilog can be done in two primary ways: Structurally or Behaviorally (Dataflow).

1. Structural Modeling

Structural Verilog is exactly like a textual schematic netlist. You instantiate primitive gates and wire them together. Consider a Half-Adder. A Half-Adder uses an XOR gate for the Sum and an AND gate for the Carry.

module half_adder_structural (
    input a,
    input b,
    output sum,
    output carry
);
    // Instantiating primitives
    xor g1 (sum, a, b);
    and g2 (carry, a, b);
endmodule

While explicit, this becomes incredibly verbose for anything larger than a few gates. Imagine writing structural Verilog for a 32-bit ALU!

2. Dataflow Modeling (Continuous Assignment)

Instead of placing individual gates, we use the assign keyword to describe how data flows from inputs to outputs using Boolean operators.

module half_adder_dataflow (
    input a,
    input b,
    output sum,
    output carry
);
    assign sum = a ^ b;
    assign carry = a & b;
endmodule

When the synthesizer reads this, it automatically infers the XOR and AND gates. The assign statement continuously drives the output wire based on the inputs.

Translating Sequential Logic: Flip-Flops and Registers

Combinational logic is only half the battle. Digital systems need memory to perform sequences of operations. This introduces Sequential Logic, primarily driven by a clock signal. The clock is the heartbeat of the FPGA, synchronizing the transfer of data between flip-flops.

In a schematic, a D-type Flip-Flop (DFF) is a block with a D (Data) input, a clock input, and a Q output. On the rising edge of the clock, the value at D is captured and pushed to Q.

In Verilog, sequential logic is modeled using always blocks triggered by edge events.

module d_flip_flop (
    input clk,
    input rst_n, // Active-low reset
    input d,
    output reg q
);
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            q <= 1'b0;
        end else begin
            q <= d;
        end
    end
endmodule

Notice the use of the non-blocking assignment operator <= instead of =. This is a critical distinction in Verilog.

Blocking vs. Non-Blocking Assignments

One of the most common pitfalls when moving from schematics to Verilog is misunderstanding assignment types within always blocks.

  • Blocking (=): Executes sequentially. The next line does not execute until the current line is finished. Used for combinational logic in always @(*) blocks.
  • Non-Blocking (<=): Executes concurrently. All right-hand side expressions are evaluated at the beginning of the clock edge, and all left-hand sides are updated simultaneously at the end of the time step. Used for sequential logic (Flip-Flops).

If you use a blocking assignment for a shift register, you will accidentally infer a single flip-flop instead of a chain, because the value propagates instantaneously rather than waiting for the next clock cycle.

Finite State Machines (FSMs) in Verilog

On paper, a Finite State Machine is drawn as a series of bubbles (states) and arrows (transitions). Converting this schematic diagram into Verilog involves a structured, three-block architecture:

  1. State Register (Sequential): An always @(posedge clk) block that updates the current state to the next state.
  2. Next State Logic (Combinational): An always @(*) block that determines the next state based on inputs and the current state.
  3. Output Logic (Combinational or Sequential): Logic that determines the outputs based on the state (Moore machine) or state + inputs (Mealy machine).
module simple_fsm (
    input clk,
    input rst,
    input btn,
    output reg led
);
    // State definitions
    parameter STATE_OFF = 1'b0;
    parameter STATE_ON  = 1'b1;

    reg current_state, next_state;

    // 1. State Register
    always @(posedge clk or posedge rst) begin
        if (rst) current_state <= STATE_OFF;
        else     current_state <= next_state;
    end

    // 2. Next State Logic
    always @(*) begin
        case (current_state)
            STATE_OFF: next_state = btn ? STATE_ON : STATE_OFF;
            STATE_ON:  next_state = btn ? STATE_OFF : STATE_ON;
            default:   next_state = STATE_OFF;
        endcase
    end

    // 3. Output Logic (Moore)
    always @(*) begin
        led = (current_state == STATE_ON);
    end
endmodule

This strict separation ensures that the synthesis tool correctly maps the FSM to the FPGA's dedicated hardware resources without introducing latches or timing loops.

Hierarchical Design and Modules

Just as schematics can encapsulate complex circuits into sub-blocks (hierarchical schematics), Verilog uses modules to manage complexity. A large FPGA design is structured as a top-level module that instantiates numerous sub-modules, which in turn instantiate smaller modules, all the way down to the gate level.

Consider integrating the ALU and Registers for a simple CPU:

module cpu_top (
    input clk,
    input rst,
    output [7:0] port_out
);
    wire [7:0] alu_result;
    wire [7:0] reg_a_out;
    wire [7:0] reg_b_out;

    // Instantiate ALU
    alu my_alu (
        .in_a(reg_a_out),
        .in_b(reg_b_out),
        .op_code(3'b001), // ADD
        .result(alu_result)
    );

    // Instantiate Registers
    register_file regs (
        .clk(clk),
        .rst(rst),
        .write_data(alu_result),
        .read_a(reg_a_out),
        .read_b(reg_b_out)
    );

    assign port_out = alu_result;
endmodule

This hierarchical approach makes code modular, reusable, and easy to verify. Modern IP (Intellectual Property) cores are distributed as highly parameterized Verilog modules that can adapt to different bit-widths and feature sets.

The FPGA Synthesis Process: From Text to Silicon

Once your Verilog code is written, how does it become a physical circuit on an FPGA? Unlike software compilers that generate machine code for a CPU, FPGA tools run a process called Synthesis and Implementation.

  1. Elaboration: The tool parses the Verilog, checks syntax, expands parameters, and builds a massive in-memory graph of the structural hierarchy.
  2. Synthesis: The behavioral descriptions (e.g., assign sum = a + b;) are translated into generic gate-level representations. The tool applies Boolean algebra to optimize the logic, minimizing gate count and critical path delay.
  3. Mapping (Technology Mapping): The generic gates are mapped to the specific hardware resources available on the target FPGA. Instead of discrete AND/OR gates, FPGAs use Look-Up Tables (LUTs). A 4-input LUT can implement ANY 4-input Boolean function.
  4. Place and Route (PAR): The tool physically places the configured LUTs and Flip-Flops onto the FPGA silicon die and computes the optimal routing pathways through the interconnect matrix to connect them. This step solves complex NP-hard layout problems to ensure all signals meet strict setup and hold timing constraints.
  5. Bitstream Generation: The final routing and LUT configuration data is packed into a proprietary binary file (the bitstream) which is then downloaded to the FPGA, physically configuring the SRAM cells that control the interconnects.

Understanding this underlying process is crucial. When writing Verilog, an expert engineer is always visualizing the resulting hardware. They know that a massive multiplexer will consume many LUTs, and that chaining too much combinational logic between flip-flops will drop the maximum clock frequency (fMax) due to long propagation delays.

Dealing with Multiple Clock Domains (CDC)

As designs grow from simple schematics to System-on-Chip (SoC) FPGA architectures, they inevitably encounter multiple clock domains. For example, a DDR memory controller might run at 400MHz, while the main processor runs at 100MHz, and a UART interface runs at a sluggish 1.8432MHz.

Crossing signals between these asynchronous clock domains is a fundamental challenge in Verilog that schematics rarely address adequately. If a signal from the 100MHz domain is sampled directly by a flip-flop in the 400MHz domain, it may violate setup or hold times, plunging the flip-flop into a metastable state where its output hovers between 0 and 1, eventually causing catastrophic system failure.

The standard Verilog solution is a 2-stage or 3-stage synchronizer for single-bit signals:

module synchronizer (
    input clk_dest,
    input async_sig,
    output synced_sig
);
    reg stage1, stage2;

    always @(posedge clk_dest) begin
        stage1 <= async_sig;
        stage2 <= stage1;
    end

    assign synced_sig = stage2;
endmodule

For multi-bit buses, FIFOs (First-In-First-Out buffers) utilizing dual-port RAM and Gray code pointers must be instantiated. Recognizing when and where to apply CDC techniques is the hallmark of a senior FPGA engineer.

Conclusion

Transitioning from schematic capture to Verilog is a profound mental shift. It requires abandoning the comfort of visual routing and embracing the power of concurrent textual descriptions. However, the benefits are immense: parameterization, version control, automated testing, and the ability to synthesize billion-transistor designs.
